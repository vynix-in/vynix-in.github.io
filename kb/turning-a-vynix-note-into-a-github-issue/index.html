Turning a Vynix note into a GitHub issue

Updated 22 June 2026

You can turn a Vynix note into a GitHub issue so your team or coding agent has the context needed to start fixing it. Vynix includes the selected element, screenshot, console and network context, and AI diagnosis where available.

  1. Open the site that has the Vynix widget installed.
  2. Click the Vynix widget, then select the page element or area that has the issue.
  3. Review the note details, including the screenshot, captured context, and AI diagnosis.
  4. Choose the option to open or create a GitHub issue.
  5. Select the repository, add or adjust the title and description, then assign it if needed.
  6. Create the issue in GitHub.

Vynix is designed to turn a visual bug report into developer-ready context. When you create a note, it captures what you clicked, what the page looked like, and useful browser context such as console and network information.

Before creating a GitHub issue, make sure your GitHub connection is set up and that you have permission to create issues in the target repository. If you do not see the GitHub option, check your workspace integration settings or review the Vynix docs.

After the issue is created, open it in GitHub to confirm the title, description, labels, assignee, and any links back to the Vynix note. You can then hand it to a developer or assign it to a coding agent.

Related questions

What information is included in the GitHub issue?

The issue can include the Vynix note details, screenshot, selected element context, console and network context, and an AI diagnosis when available. The exact fields may depend on your workspace settings.

Why do I not see the option to create a GitHub issue?

Your workspace may not have GitHub connected, or your account may not have permission to create issues in the selected repository. Check your integration settings or see the Vynix docs at https://vynix.in/docs.

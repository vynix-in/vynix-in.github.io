How to install the Vynix widget on any website

Updated 29 June 2026

When something is broken on a website, the hardest part is often explaining exactly what happened and where. Vynix adds a lightweight annotation widget to your site so your team can click the problem, capture the page context, and turn it into a clear developer task.

  1. Create or open your Vynix project. Go to the Vynix app, choose the project you want to connect to your website, and open the installation or widget setup area. If you have not created a project yet, create one for the site or environment you want to test, such as production, staging, or a local preview.
  2. Copy the widget install snippet. Vynix provides a small script snippet for your project. Copy the full snippet exactly as shown. The snippet usually includes the project or workspace information Vynix needs to connect captured annotations back to the right place.
  3. Paste the snippet into your website. Add the script to your site layout so it loads on every page where you want Vynix to be available. For many websites, this means pasting it before the closing body tag in your main HTML file or shared layout component. For a CMS or site builder, use the custom code, header, footer, or tracking script area. For a web app, add it to the root layout, document template, or main HTML shell.
  4. Save, publish, or redeploy your site. After adding the snippet, save your changes and deploy them to the environment you want to use. If you are testing locally, restart your dev server if needed. If you are using a hosted platform, publish the change and wait for the build or cache refresh to finish.
  5. Open the site and confirm the widget appears. Visit a page where the script should load. You should see the Vynix widget or launcher based on your project settings. If it does not appear, hard refresh the page, check that the script is present in the page source, and confirm that the domain or environment is allowed in your Vynix settings.
  6. Try a test annotation. Click the widget, select something on the page, and submit a short note such as "Test annotation". Vynix should capture the selected element, a screenshot, browser console context, network context, and an AI diagnosis of the likely root cause. Open the captured item in Vynix to confirm the details look correct.
  7. Connect your issue workflow if needed. If your team uses GitHub, connect the integration from Vynix and choose how issues should be created. Once connected, you can turn an annotation into a GitHub issue, include the captured context, and assign it to the right person or coding agent.

Vynix works by loading a small client-side widget on your website. When someone opens the widget and clicks on a broken or confusing part of the page, Vynix records the selected element and the visible page state. It also collects useful developer context, such as recent console output and network activity, so the person fixing the issue does not have to ask for screenshots, browser details, or reproduction steps before they can start.

The main benefit is that feedback becomes tied to the actual page element that needs attention. For example, imagine a checkout page where the "Apply discount" button looks active but does nothing. Instead of writing a vague bug report, a teammate can click that exact button with Vynix, add a note like "Discount code does not apply", and submit it. Vynix captures the button, the screenshot, the network request that failed or did not fire, and an AI diagnosis that may point to a missing handler, validation issue, or backend response problem. From there, the team can copy a ready-to-build prompt or open a GitHub issue with the context already attached.

Install Vynix in the place where shared scripts belong for your website. On a static HTML site, this is usually a shared template or each page that needs the widget. In a React, Vue, Next.js, Nuxt, SvelteKit, or similar app, add it to the app shell or root layout so it is loaded once. In WordPress, Webflow, Shopify, or another CMS, use the platform's custom code area if available. The exact location depends on your stack, so check your framework or platform documentation if you are unsure.

A common mistake is adding the snippet to only one page, then expecting Vynix to appear everywhere. If your site uses separate layouts for marketing pages, app pages, checkout, docs, or account screens, make sure the snippet is added to each layout where annotations are needed. Another common mistake is testing too quickly after deployment. Some hosting platforms and CDNs cache pages, so the old version of your site may still be served for a short time. Use a hard refresh, private browsing window, or cache purge if the widget does not show up after publishing.

If the widget still does not load, check for script blockers, content security policy rules, and environment restrictions. Some sites only allow scripts from approved domains, so your security policy may need to allow the Vynix widget source. Some projects also restrict which domains can use the widget, so confirm that your current domain, staging URL, or local development URL is allowed. For the latest setup details and platform-specific guidance, see the Vynix docs at https://vynix.in/docs.

Related questions

How do I add Vynix to my site?

Create or open a Vynix project, copy the widget install snippet, paste it into your website's shared layout or custom code area, then publish your site. After publishing, open the site and submit a test annotation to confirm Vynix is capturing the page, screenshot, console context, and network context correctly.

Can I install Vynix on any website?

Vynix is designed to work on any website where you can add a JavaScript widget. This includes custom web apps, static sites, CMS pages, and many no-code or low-code builders. If your platform limits custom scripts or has strict security settings, check the platform settings and the Vynix docs for the recommended setup.

Why is the Vynix widget not showing after I installed it?

First, confirm the snippet was saved and published to the page you are viewing. Then hard refresh the page and check that the script appears in the page source. If it still does not load, review domain allowlists, content security policy settings, browser extensions that block scripts, and any caching from your hosting provider or CDN.
